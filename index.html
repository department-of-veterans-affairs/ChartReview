Features and Requirements




First Class Design

Designed with the enterprise in mind, the codebase is robust, extendable, scalable, and unit tested.

Access YOUR Data

Chart Review should support most JDBC data sources, though is currently only tested against MySQL and Microsoft SQL Server.  How a patient is defined, and what data elements make up particluar medical records are configured by the user in the application. This provides unparallelled views into your existing medical database withtout requiring database or workflow changes.

Requirements

ChartReview requires:

  • A Java Servlet Container. Tomcat is recommended, but any standard Servlet Container 3.0+ is supported.
  • A Mysql Database for holding ChartReview metadata
  • A JDBC accessible medical record database.
